
## skill 分类

| 场景                           | 推荐模式       | 你需要做的                                  |
| :----------------------------- | :------------- | :------------------------------------------ |
| 教 LLM 遵循某种规范/风格       | 纯 Prompt 注入 | 写好 SKILL.md，全靠文本质量                 |
| 需要 LLM 操作特定文件格式      | 脚本执行型     | 写预制脚本放 scripts/，SKILL.md 写使用说明  |
| 需要 LLM 灵活组合 API          | 库调用型       | 写 Python 库放 core/，SKILL.md 当 API 文档  |
| 知识量大，不同任务需要不同文档 | 渐进加载型     | SKILL.md 做路由表，详细文档独立存放         |
| 需要 LLM 执行复杂多步骤工作流  | 编排型         | SKILL.md 定义流水线 + 脚本工具链 + 子 Agent |


技能的设计和 agent 设计是类似的，



把 Agent 当成一个 "会自己做决策的系统" 来看。
1. 先看基础能力，比如它能不能正确理解指令、拆解任务、选对工具，推理链路对不对。
2. 然后是稳定性，同样的问题问三遍，答案会不会飘得离谱。
3. 再往深了说，还得测边界场景，比如给它模糊指令、矛盾信息，看它怎么处理，异常情况能不能兜住。
4. 最后是安全性，这个现在越来越重要，得确保它不会被 prompt injection 攻击，也不会输出不该输出的东西。
5. 如何设计测试用例。
自测Agent核心需要明确：
1. 要做什么？
2. 为什么这么做？
3. 怎么证明这么做是对的？
