# 制作一个提示词优化Skill

## step 1
判断任务级别。
通过对用户提问 "你愿意为这个任务花多少时间？" 来判断任务级别。
- 选项1: < 5分钟
- 选项2: 2小时左右
如果 < 5分钟，则 给例子+定格式；
如果 2小时左右，则 按照流程提问完善提示词；

### 输入格式化
用 XML 标签做结构。把上下文资料、任务说明、输出格式分别用 <document>（资料）、<task>（任务）、<format>（格式）这种标签包起来。
标签前后一致就行。

### 输出格式
用户确认输出格式
- 选项1: 列表
- 选项2: 表格
- 选项3: JSON
- 选项4: markdown
- 选项5: 自定义

### 输出要求

直接、明确说要什么，不要绕弯子。

### 复杂任务
复杂任务前面写一句「先一步步分析，再给最终答案」。
- 基础（直接写一句「先一步步思考」）
- 引导（列出具体的分析步骤）
- 结构化（用 <thinking>（思考）和 <answer>（回答）这种标签把「想」和「答」分开）。

按照以下流程提问完善提示词：
1. 让 AI 自选是否要必要角色
2. 让 AI 连续提问，直到 95% 理解
   1. 不知道如何界定是哪个领域的问题，让 AI 自选角色，“你们认为解决这个问题，最需要哪三类专家角色？我们先按这个角色分工来脑暴。”
   2. 不知道自己是否清楚自己是否已经完全搞懂问题，增加一步："在行动前，请向我连续提问，直到你 95%确信理解我的目标和边界。"
3. 先跑一个具体情境。
4. 根据表现迭代
5. 让“红队”挑刺。成立一个小组，唯一任务就是挑这个方案的毛病。“假设这个[项目]彻底失败，现在复盘，最可能的原因是什么？”

## step 2
判断任务场景。

1. 如果是创意写作、策略建议、枚举类问答(有多个合理答案的场景)。需要添加候选内容相关的提示词。
```text
1. 生成 5 个不同的候选答案（内容尽量有风格差异）。
2. 为每个答案给出 0-1 概率，并保证总和为 1。// 必要时提示“概率必须非负”。
3. 按概率从高到低排序，逐条输出为：内容｜概率。
4. 在末尾给出“采样建议”：
   - 若只需 1 个答案，推荐选择概率最高者；
   - 若需多样性，可按概率加权进行随机采样。
```

2. 如果写作、风格化这类场景，添加角色扮演型提示词："你是一个专业的[xxxxx]专家，负责解决[xxxxx]问题。"如果是做事实任务则不要添加。
